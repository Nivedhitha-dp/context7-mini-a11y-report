Expand/Collapse Widget Examples

Example 1: Basic Information

An expand/collapse widget is an interactive UI component that allows users to show or hide content on demand. This helps manage screen space and reduces cognitive load by allowing users to access information progressively.

These widgets are commonly used in FAQs, navigation menus, and content-heavy pages where not all information needs to be visible at once.

Example 2: Accessibility Features

Accessibility ensures that all users, regardless of their abilities or disabilities, can access and use web content effectively. This includes users who rely on keyboard navigation, screen readers, or other assistive technologies.

Proper implementation includes using semantic HTML, ARIA attributes, keyboard support, and sufficient color contrast ratios to meet WCAG 2.2 Level AA standards.

Example 3: Keyboard Navigation

Tab: Move focus to the expand/collapse button

Enter or Space: Toggle the content visibility

Shift + Tab: Move focus backward

All interactive elements are fully accessible via keyboard, ensuring users who cannot use a mouse can still interact with the content.

Example 4: Implementation Details

Semantic HTML: Uses a <button> element for interaction, which is natively keyboard accessible and recognized by assistive technologies.

ARIA Attributes: The aria-expanded attribute communicates the current state (expanded or collapsed) to screen readers, while aria-controls creates a relationship between the button and the content it controls.

Focus Management: Visible focus indicators ensure users can see which element is currently focused.

Visual Indicators: The arrow icon rotates to provide a visual cue about the expanded/collapsed state.
