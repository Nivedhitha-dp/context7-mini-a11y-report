Accordion Pattern

An accordion is a vertically stacked set of interactive headings that each reveal a section of content. Only one panel is typically expanded at a time, though some implementations allow multiple panels to be open simultaneously.

Accordions are useful when you have multiple sections of content on a page and want to keep the page length manageable. They work well for:

  • Frequently Asked Questions (FAQs)
  • Product features or specifications
  • Step-by-step instructions
  • Settings or configuration options

To create accessible accordions:

  • Use button elements for the interactive headers
  • Include aria-expanded to indicate state
  • Use aria-controls to associate buttons with panels
  • Wrap button text in heading elements for proper document structure
  • Ensure keyboard operability (Enter/Space to toggle)
  • Provide visible focus indicators
  • Consider adding arrow key navigation between accordion headers

Common accessibility mistakes with accordions include:

  • Using div or span elements instead of buttons
  • Forgetting to update aria-expanded when toggling
  • Not providing sufficient color contrast for text and icons
  • Making the entire panel header clickable instead of just the button
  • Hiding content with CSS in a way that makes it still keyboard-focusable
  • Not managing focus properly when panels open/close
