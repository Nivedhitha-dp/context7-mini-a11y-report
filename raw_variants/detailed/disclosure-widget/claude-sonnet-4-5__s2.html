Expand/Collapse Widget Examples

These widgets demonstrate accessible expand/collapse patterns using buttons with aria-expanded.

Example 1: Basic Expand/Collapse

An expand/collapse widget, also known as a disclosure widget, allows users to show or hide content on demand. This helps reduce cognitive load by presenting only the most relevant information initially.

The button uses aria-expanded to communicate its state to assistive technologies, and the content is associated using aria-controls.

Example 2: FAQ Section

To make an expand/collapse widget accessible:

  • Use a <button> element for the control
  • Add aria-expanded to indicate the current state (true or false)
  • Use aria-controls to associate the button with the content region
  • Ensure the button has a visible label that describes what will be expanded
  • Make sure keyboard users can operate it (Enter and Space keys)
  • Provide a visible focus indicator

Example 3: Additional Information

The HTML <details> and <summary> elements provide native expand/collapse functionality and are generally recommended for simpler use cases.

However, custom implementations may be necessary when:

  • You need more control over animations and transitions
  • The design requires custom styling that's difficult to achieve with <details>
  • You need to coordinate multiple widgets (like an accordion where only one can be open)
  • Browser support for <details> doesn't meet your requirements

When building custom widgets, always ensure they meet WCAG accessibility requirements.
