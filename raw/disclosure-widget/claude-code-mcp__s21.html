Accessibility (a11y) means designing and building digital products so that people with disabilities — including those using screen readers, keyboard navigation, or switch controls — can use them fully and independently.

Semantic HTML gives browsers and assistive technologies the right context for each element. A <button> is automatically keyboard-operable and announced correctly by screen readers — something a styled <div> with an onclick never provides without extra ARIA work.

WCAG 2.1 Level AA is the internationally recognised standard for web accessibility, covering perceivability, operability, understandability, and robustness. Meeting AA is the legal baseline in many jurisdictions and a strong signal of inclusive design practice.
