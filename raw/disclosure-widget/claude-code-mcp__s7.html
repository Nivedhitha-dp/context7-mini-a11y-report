WCAG stands for Web Content Accessibility Guidelines. It is published by the World Wide Web Consortium (W3C) as part of the Web Accessibility Initiative (WAI).

The guidelines provide a single shared standard for web content accessibility that meets the needs of individuals, organizations, and governments internationally.

Many users rely exclusively on a keyboard to navigate the web — including people with motor disabilities, power users, and those using assistive technologies like switch access.

All interactive elements must be reachable and operable using Tab, Enter, Space, and arrow keys without requiring a pointer device.

Accessible Rich Internet Applications (ARIA) is a set of roles and attributes that define ways to make web content and web applications more accessible to people with disabilities.

The first rule of ARIA is: if you can use a native HTML element or attribute with the semantics and behavior you require already built in, then do so instead of using ARIA.

Use a combination of automated tools (Axe, Lighthouse, WAVE) and manual testing. Manual testing should include keyboard-only navigation and at least one screen reader (NVDA, JAWS, VoiceOver).

Automated tools catch roughly 30–40% of issues — manual testing is essential for the rest.
