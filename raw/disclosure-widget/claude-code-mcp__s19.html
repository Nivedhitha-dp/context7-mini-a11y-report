Accessible web design means creating websites and applications that can be used by people with a wide range of abilities and disabilities. It follows WCAG guidelines to ensure content is perceivable, operable, understandable, and robust.

Many users navigate the web exclusively via keyboard — including people with motor disabilities, power users, and those using assistive technologies like screen readers. Every interactive element must be reachable and operable without a mouse.

ARIA (Accessible Rich Internet Applications) is a set of attributes that supplement HTML semantics for assistive technologies. The first rule of ARIA is to use native HTML elements whenever possible — only reach for ARIA when no native equivalent exists.
